사이드 프로젝트

회사 사람들끼리 쓸 정도의 간단한 메신저 프로그램을 만들어 보려고 한다.

  1. 메신저 프로그램을 만드는 이유는 평소 업무상 의사소통을 카카오톡을 이용하는데, 내PC는 리눅스 운영체제를 사용중이다.
    카카오톡은 현재 리눅스 운영체제 버전의 응용프로그램이 서비스 중이지 않고, 난 그것이 많이 불편하다.

  2. 현재의 목표는 웹 페이지에서 동작하는 채팅 프로그램을 만드는 것. 그 이후에 데스크탑에서 동작하는 응용프로그램, 스마트폰에서 동작하는 앱을 만들어 사용하는 것이다.

    • 메신저 웹 페이지
    • 데스크탑 응용프로그램
    • 스마트폰 응용프로그램

  3. 메신저 프로그램을 만드는 것에 대해서 무엇을 사용하여, 어떻게 만들지를 이렇게 생각했다.

    • 메신저 웹 페이지: Front-End는 React를, Back-End 생각해두지 않았다.(아마 Spring Boot나 Node JS를 사용하지 않을까?)
      React는 페이스북, 네이버, 인스타그램 등 유명한 회사에서도 사용 중인 라이브러리이다.
      커뮤니티의 규모가 크고 넓으며, 다양한 자료도 얻을 수 있다고 한다.

    • 데스크탑 응용프로그램: Electron에 React 프레임워크로 작성한 페이지를 올려 사용할 생각이다.
      Electron은 Visual Studio Code, Atom, Discord와 같은 프로그램이 Electron을 사용하여 만들어졌다고 한다.
      실제로 이 깃헙 페이지도 VS Code를 사용하여 작성 중이며, 웹으로 만들어졌다고 생각되지 않을정도로 빠르고 매우 편리하다.

    • 스마트폰 응용프로그램: React-Native로 만들 생각이다. 전에 안드로이드 어플을 만들 떄 웹 뷰 컴포넌트를 통해서 화면을 띄울떄는 성능의 저하가 매우 심하게 일어났었다.
      React-Native로 만들어진 상용 프로그램 중에는 에어 비앤비, 인스타그램 등이 있다고 한다.
      에어 비앤비와 인스타그램이 사용해 봤을 때 속도가 느리지 않다고 느껴지기 때문에 React-Native를 사용하여 만들어볼 생각이다.

마무리 - 사이드 프로젝트는 여기까지만 생각을 했고, React를 시작하는것은 생활코딩의 강좌를 보고 시작하려 한다.
기초 적인 것을 보고 따라하면서 웹 프로그램을 띄우는 것이 현재의 단기적인 목표이다.
