2020-08-07

TDD 책을 읽고 난 뒤, 난 이것을 어떻게 실천할지 모르겠다.

난 2년 반동안 소프트웨어 개발이라는 일을 하면서, 큰 몇가지의 문제를 겪고있었다.

항목
존재하지 않는 문서
잦은 요구사항 변경
신규 기능 구현과 같이 진행되는 유지보수
누군사 코드에 손을 대면 작동이 안되는 기존의 기능

신규 기능 구현과 같이 진행되는 유지보수과 누군사 코드에 손을 대면 작동이 안되는 기존의 기능이 두가지를 해결하고 싶었다. 그래서 TDD(Test-Driven Development:By Example, 켄트 백 지음)책을 읽었다.

처음 읽을 때에는 저자가 말하고자 하는 바를 이해하려했다.
읽기만 하고 끝나니 기억에 남는 것이 별로 없었다. 빨간막대 패턴? 초록막대 패턴? 이 두 가지만 남았다.
두번째는 예제를 읽고 타이핑을 했다. 코드나 리비전에 했던것이 남아있으니, 그리고 손에 뭔가 남는 느낌이 들었다.

두 번을 읽고 나서 이제 내가 만들고 싶은 기능에 TDD를 적용하고 싶었다.
너무 답답했다. 어디까지나 예제를 보고 타이핑하는 것은 타이핑에서 끝나는 것인가 보다.
내 생각이 테스트 주도적으로 개발을 하도록 바뀐 것은 아니었다.

이 책은 좋은 책이 맞고, 언젠가 내가 테스트 주도적으로 개발을 할 수 있는 날을 상상하면 기분이 좋다.
현재는 막힐 때마다 참고서의 풀이를 읽고 문제를 해결하듯이, 내가 하고 싶은것을 잘 안풀릴 때마다 이 책을 읽고 행동으로 취해봐야겠다.

Sample Application- PersonnelView

목차

  1. ExtjsCe Quick Start
  2. Sapmle Application- 실행화면
  3. Sample Application- HomeView


1. PersonnelView.js의 코드

Ext.define 명령어를 사용하여 PersonnelView에 대한 class를 정의한다. config는 다음과 같다.
xtype :personnelview’ (homeview라는 컴포넌트를 생성할때 ‘MyExtGenApp.view.personnel.PersonnelView’또는 ‘personnelview’만을 적어서 생성 할 수 있다.)
cls: ‘personnelview’ (personnelview라는 이름을 가진 class[html태그의 class 선택자]를 추가한다.)
contoller: ‘personnelviewcontroller’ (personnelviewcontroller를 controller로 사용.)
viewmodel: ‘personnelviewmodel’ (personnelviewmodel을 viewmodel로 사용한다.)
store: {type: ‘personnelviewstore’} (personnelviewstore를 사용함)
extend: ‘Ext.grid.Grid’ (Ext.grid.Grid를 상속받는다, Ext.gird.Grid는 DataGrid의 형태를 가지고 있다.)
columns: [
    { text: ‘Name’,
      dataIndex: ‘name’,
      width: 100,
      cell: {userCls: ‘bold’}
    }
]
: (데이터 그리드의 컬럼(열)을 생성하는 방법이다.
text: ‘Name’의 컬럼명을 가진,
dataIndex: ‘name’ personnelviewstore의 name key의 값을 행의 내용에 넣는다.
열의 너비는 width: 100만큼 차지하고,
cell: {userCls: ‘bold’} 셀의 특정한 속성을 넣는다.)

listeners: {
    select: ‘onItemSelected’
}
: (데이터 그리드의 행을 선택(select) 할 때 personnelviewcontorller에 정의된 onItemSelected 함수를 실행하는 이벤트 처리에 대하여 정의되어 있다.)

2. PersonnelViewController.js의 코드

Ext.define 명령어를 사용하여 PersonnelViewController에 대한 Class를 정의한다. config는 다음과 같다.
alias :’controller.personnelviewcontroller’ (컨트롤러 생성할때 {type: ‘homeviewcontroller’}만을 적어서 생성 할 수 있다.)
extend: ‘Ext.app.ViewController’ (Ext.app.ViewController를 상속받는다.)

onItemSelected: function (sender, record) {
    Ext.Msg.confirm(‘confirm’, ‘Are you sure?’, ‘onConfirm’, ‘this);
}: personnelview에 생성된 레코드를 선택했을때 실행되는 함수이다. (sender와 record를 argument로 정의되어 있다.)
Ext.Msg.confirm 기능을 실행하여 Are you sure? 라고 물어보는 모달창을 생성하는 함수이다. 모달창에는 YES와 NO를 선택할 수 있는 버튼이 있는데 둘중 하나를 선택하면 onConfirm 함수를 실행하는 이벤트가 정의되어 있다.

onConfirm: function (choice) {
    if (choice === ‘yes’) {
                //
   }
}: onItemSelected에서 생성된 모달창의 버튼을 눌렀을 때 실행되는 함수이다. argument중 choice가 버튼에 대한 값이 들어간다. 함수의 내용에는 yes버튼을 눌렀는지 체크하는 조건문이 있다.

3. PersonnelViewModel.js의 코드

Ext.define 명령어를 사용하여 PersonnelViewModel에 대한 Class를 정의한다. config는 다음과 같다.
alias :’controller.personnelviewmodel’ (뷰 모델 생성할때 {type: ‘personnelviewmodel’}만을 적어서 생성 할 수 있다.)
extend: ‘Ext.app.ViewModel’ (viewmodel class 를 상속받는다.)

4. PersonnelStore.js의 코드

Ext.define 명령어를 사용하여 PersonnelViewStore에 대한 Class를 정의한다. config는 다음과 같다.
alias :’store.personnelviewstore’ (persoennlview의 store를 생성할때 {type: ‘personnelviewstore’}만을 적어서 생성 할 수 있다.)
extend: ‘Ext.data.Store’ (Store class 를 상속받는다.)
field: [
    ‘name’, ‘email’, ‘phone’
]: (해당 스토어의 field가 정의되어 있다. 이 코드에서는 총 3가지 필드가 정의되어 있다.)
data: {
    items: [
        { name: ‘Jean Luc’, email: “jeanluc.picard@enterprise.com“, phone: “555-111-1111” },
    ]
}: (스토어 내 레코드 정보가 정의도어 있다. Array형태로 여러 줄 만들 수 있다.)
proxy: {
    type: ‘memory’,
    reader: {
        type: ‘json’,
        rootProperty: ‘items’
    }
}: (메모리 내 프록시를 사용한다. 페이지를 새로고침 할 때마다 내용은 손실된다. jsonReader로 MemoryProxy를 사용하여 items의 내용을 읽어온다.)

PersonelView.scss 파일은 열어보았지만 코드가 없어서 생략한다.
다음 포스트는 mainview에 대하여 다뤄야겠다.

Sample Application- HomeView

샘플 프로젝트의 폴더는 밑의 사진처럼 구성되어 있다. View folder의 하위에 home, main, personnel이 존재하며

home~ personnel 폴더의 하위에 화면을 구성하는데 필요한 폴더 혹은 파일이 위치해있다.

home과 personnel 폴더는 각 homeview와 personnelview를 구성하는 파일을 구성하는 폴더이며,
main폴더는 Sample Application을 구성하는 구조를 정의해놓은 파일 및 폴더의 집합이다.

home 폴더를 살펴보자. 화면의 모습을 보여줄 HomeView.js 파일과 기능 등을 담당할 HomeViewController.js 그리고 HomeView에 대한 Meodel인
HomeViewMode가 있다. HomeView의 스타일을 꾸미고 싶은 경우 HomeView.scss에 코드를 작성하면 된다.

1. HomeView.js의 코드

Ext.define 명령어를 사용하여 HomeView에 대한 Class를 정의한다. config는 다음과 같다.
xtype :’homeview’ (homeview라는 컴포넌트를 생성할때 ‘MyExtGenApp.view.home.HomeView’또는 ‘homeview’만을 적어서 생성 할 수 있다.)
cls: ‘homeview’ (homeview라는 이름을 가진 class[html태그의 class 선택자]를 추가한다.)
contoller: ‘homeviewcontroller’ (homeviewcontroller를 controller로 사용.)
viewmodel: ‘homeviewmodel’ (homeviewmodel을 viewmodel로 사용한다.)
extend: ‘Ext.Container’ (Ext.Continer를 상속받는다.)
scrollable: true (해당 컴포넌트는 스크롤이 가능하다.)
html: ‘<div style=”user-select:…’ (는 해당 컴포넌트의 html 태그)

2. HomeViewController.js의 코드

Ext.define 명령어를 사용하여 HomeViewController에 대한 Class를 정의한다. config는 다음과 같다.
alias :’controller.homeviewcontroller’ (컨트롤러 생성할때 {type: ‘homeviewcontroller’}만을 적어서 생성 할 수 있다, xtype과 alias에 대한 부분은 좀 더 조사하여 나중에 다루도록 하겠음.)
extend: ‘Ext.app.ViewController’ (Ext.app.ViewController를 상속받는다.)

3. HomeViewModel.js의 코드

Ext.define 명령어를 사용하여 HomeViewModel에 대한 Class를 정의한다. config는 다음과 같다.
alias :’controller.homeviewmodel’ (뷰 모델 생성할때 {type: ‘homeviewmodel’}만을 적어서 생성 할 수 있다, xtype과 alias에 대한 부분은 좀 더 조사하여 나중에 다루도록 하겠음.)
extend: ‘Ext.app.ViewModel’ (homeviewmodel를 상속받는다.)

4.HomeView.scss의 코드

사실 scss에 대한것은 잘 모른다. 하지만 알고 있는 부분만 풀어놓고자 한다. (sass에 대한것도 alias 및 xtype과 같이 조사하여 나중에 다루도록 하겠다.)

  1. HomeView.scss라는 파일의 이름은 home폴더의 하위에 있는 HomeView.js와 같은 이름의 규칙을 지켜서 만들어야 한다. HomeNotView와 같이 다른 이름을 가진 scss파일을 만든 후
    style을 정의해보았지만 적용되지 않았었다.
  2. .homeview는 HomeView.js에서 정의한 cls: ‘homeview’ 구성과 같다. 실제 HomeView 컴포넌트를 구성하고 있는 Html 태그를 살펴보면 homeview class선택기가 추가되어 있다.
  3. color와 background-color는 sass 를 사용하여 미리 정의한 $color와 $base-foreground-color 변수 값을 받아서 사용하고 있다.
  4. font-size: 18px (homeview 컴포넌트 안에서 글씨들은 18px만큼의 크기를 가진다.)
  5. padding: 15px 0px 0px 15px (homeview 컴포넌트 안에서 모든 글씨 혹은 그림 등은 위와 오른쪽으로 15px만큼 여백을 가지고 있다.)
  6. border: 1px solid lightgray (homeview의 바깥에 1px만큼의 굵기를 가진 lightgray색의 실선을 테두리로 만든다.)
  7. box-shadow: 2px 2px 8px 0 rgba(0, 0, 0, 0.4) (homeview의 테두리를 감싼 그림자 효과를 만든다. offset-x | offset-y | blur-radius | spread-radius | color 순서로 정의한다.)

다음은 포스트는 personnelview를 살펴볼 생각이고 personnelview의 설명할 내용이 적다면 mainview에 대한 것도 추가하여 다뤄봐야겠다.

Extjs- [require]

ExtJs Application을 만들 떄 Sencha CMD를 사용하면 <script src= “Ext-all.js” 또는 “Ext-all-debug.js”를 하는 것보다 로드 속도가 빠르게 만들 수 있다.
이것 말고도 Sencha CMD를 사용하여 모델, 스토어, 뷰 생성은 물론이고 웹서버를 띄운다거나 개발용코드 테스트용코드 배포용코드 등 여러가지 편리한 기능을 명령어로 실행시킬 수 있다.

나는 Application을 만들 때 Sencha CMD를 사용하여 만들고, 최근 require떄문에 자주 곤란한 사항에 빠지고는 한다.
그래서 이번 포스팅의 주제는 Application 안에 Extjs의 클래스를 사용할 때, requires 안에 해당 클래스를 명시하는것이 중요함을 알리고자 한다.

예를들어 탭바를 생성하는 클래스를 선언하다고 치자.

Ext.panel.Panel 컴포넌트 안에 tabbar 컴포넌트가 생성되고 tab1, tab2, tab3 탭이 순서대로 생성될 것이다.
문제는 여기서 부터 발생하는데, 개발을 할 떄는 화면이 잘 나오고 기능도 예상한대로 실행이 되니까 Sencha app build production 명령어를 실행시켜 배포용 코드를 만들어서 배포를 하고나서 웹 페이지를 접속해보면 나를 맞이하고 있는 빈 화면과 콘솔에 다음과 같은 메시지가 출력이 된다.

이러한 메시지가 나타나는 이유는 위에서 언급한 require를 하지 않았기 때문이고, 이렇게 tabbar 클래스를 사용하겠다고 requires안에 ‘Ext.tab.Bar’라고 알려주면, 배포할 경우에 아까 같은 404 에러가 나타나지 않을 것이다.

내 경우에는 무슨 404에러? 이러다가 엄청 헤멨었고, 이 글을 보는 분들은 이런 문제를 잘 넘어갔으면 좋겠다.

그리고 production하기 전에 sencha app build testing 명령어를 실행시켜 테스트용 코드를 생성하여 실행하면 어떤 문제가 있는지 쉽게 알 수 있으므로, sencha app build testing => sencha app build production 순서로 테스팅 코드를 먼저 확인하고 production을 하여 배포 코드를 확인하기 바란다.

사이드 프로젝트

회사 사람들끼리 쓸 정도의 간단한 메신저 프로그램을 만들어 보려고 한다.

  1. 메신저 프로그램을 만드는 이유는 평소 업무상 의사소통을 카카오톡을 이용하는데, 내PC는 리눅스 운영체제를 사용중이다.
    카카오톡은 현재 리눅스 운영체제 버전의 응용프로그램이 서비스 중이지 않고, 난 그것이 많이 불편하다.

  2. 현재의 목표는 웹 페이지에서 동작하는 채팅 프로그램을 만드는 것. 그 이후에 데스크탑에서 동작하는 응용프로그램, 스마트폰에서 동작하는 앱을 만들어 사용하는 것이다.

    • 메신저 웹 페이지
    • 데스크탑 응용프로그램
    • 스마트폰 응용프로그램

  3. 메신저 프로그램을 만드는 것에 대해서 무엇을 사용하여, 어떻게 만들지를 이렇게 생각했다.

    • 메신저 웹 페이지: Front-End는 React를, Back-End 생각해두지 않았다.(아마 Spring Boot나 Node JS를 사용하지 않을까?)
      React는 페이스북, 네이버, 인스타그램 등 유명한 회사에서도 사용 중인 라이브러리이다.
      커뮤니티의 규모가 크고 넓으며, 다양한 자료도 얻을 수 있다고 한다.

    • 데스크탑 응용프로그램: Electron에 React 프레임워크로 작성한 페이지를 올려 사용할 생각이다.
      Electron은 Visual Studio Code, Atom, Discord와 같은 프로그램이 Electron을 사용하여 만들어졌다고 한다.
      실제로 이 깃헙 페이지도 VS Code를 사용하여 작성 중이며, 웹으로 만들어졌다고 생각되지 않을정도로 빠르고 매우 편리하다.

    • 스마트폰 응용프로그램: React-Native로 만들 생각이다. 전에 안드로이드 어플을 만들 떄 웹 뷰 컴포넌트를 통해서 화면을 띄울떄는 성능의 저하가 매우 심하게 일어났었다.
      React-Native로 만들어진 상용 프로그램 중에는 에어 비앤비, 인스타그램 등이 있다고 한다.
      에어 비앤비와 인스타그램이 사용해 봤을 때 속도가 느리지 않다고 느껴지기 때문에 React-Native를 사용하여 만들어볼 생각이다.

마무리 - 사이드 프로젝트는 여기까지만 생각을 했고, React를 시작하는것은 생활코딩의 강좌를 보고 시작하려 한다.
기초 적인 것을 보고 따라하면서 웹 프로그램을 띄우는 것이 현재의 단기적인 목표이다.
